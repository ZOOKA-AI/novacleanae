# 🤝 دليل المساهمة | Contributing Guide

شكراً لاهتمامك بالمساهمة في Nova Clean AE! 🎉

Thank you for your interest in contributing to Nova Clean AE! 🎉

## 🇸🇦 باللغة العربية

### كيفية المساهمة

1. **Fork المستودع**
   - انقر على زر Fork في أعلى الصفحة

2. **استنساخ المستودع**
   ```bash
   git clone https://github.com/YOUR_USERNAME/novacleanae.git
   cd novacleanae
   ```

3. **إنشاء فرع جديد**
   ```bash
   git checkout -b feature/amazing-feature
   ```

4. **إجراء التغييرات**
   - اتبع معايير الكود
   - أضف تعليقات واضحة
   - اختبر التغييرات

5. **Commit التغييرات**
   ```bash
   git commit -m "إضافة ميزة رائعة"
   ```

6. **Push إلى GitHub**
   ```bash
   git push origin feature/amazing-feature
   ```

7. **إنشاء Pull Request**
   - افتح Pull Request من فرعك إلى `main`

### معايير الكود

- استخدم TypeScript بشكل صحيح
- اتبع قواعد ESLint
- نسق الكود باستخدام Prettier
- أضف تعليقات للكود المعقد
- اكتب اختبارات للميزات الجديدة

### رسائل Commit

استخدم رسائل واضحة ومفهومة:
- `feat: إضافة ميزة جديدة`
- `fix: إصلاح مشكلة`
- `docs: تحديث الوثائق`
- `style: تحسين التنسيق`
- `refactor: إعادة هيكلة الكود`
- `test: إضافة اختبارات`

## 🇬🇧 In English

### How to Contribute

1. **Fork the repository**
   - Click the Fork button at the top of the page

2. **Clone the repository**
   ```bash
   git clone https://github.com/YOUR_USERNAME/novacleanae.git
   cd novacleanae
   ```

3. **Create a new branch**
   ```bash
   git checkout -b feature/amazing-feature
   ```

4. **Make your changes**
   - Follow code standards
   - Add clear comments
   - Test your changes

5. **Commit your changes**
   ```bash
   git commit -m "Add amazing feature"
   ```

6. **Push to GitHub**
   ```bash
   git push origin feature/amazing-feature
   ```

7. **Create Pull Request**
   - Open a Pull Request from your branch to `main`

### Code Standards

- Use TypeScript properly
- Follow ESLint rules
- Format code with Prettier
- Add comments for complex code
- Write tests for new features

### Commit Messages

Use clear and descriptive messages:
- `feat: add new feature`
- `fix: fix bug`
- `docs: update documentation`
- `style: improve formatting`
- `refactor: refactor code`
- `test: add tests`
